# ①課題名
TriArk お気に入りホスト管理アプリ（ブックマークアプリ その4 / ログイン機能追加）

## ②課題内容（どんな作品か）
気になるホストや体験をブックマークして管理できるCRUDアプリに、
ログイン機能とユーザー登録機能を追加しました。
ログインの有無でアクセスできるページを分けており、
一覧の閲覧は誰でも可能、編集・削除・登録はログインしたユーザーのみが行えます。

主な機能：
- 一覧表示（select.php）… ログイン不要、誰でも閲覧可能
- ユーザー登録（register.php / register_act.php）… パスワードをハッシュ化して保存
- ログイン・ログアウト（login.php / login_act.php / logout.php）
- 登録・編集・更新・削除（insert / detail / update / delete）… ログイン必須

## ③アプリのデプロイURL
無し

## ④アプリのログイン用IDまたはPassword（ある場合）
新規ユーザー登録画面から作成可能

## ⑤工夫した点・こだわった点
- パスワードは password_hash() でハッシュ化して保存し、ログイン時は
  password_verify() で照合することで、平文でパスワードを保存しない安全な作りにしました。
- ログインの有無でページを場合分けし、一覧は誰でも見られる一方、
  編集・削除などデータを変更する操作はログインユーザーのみに限定しました。
- ログイン処理では life_flg=0（有効なユーザー）のみを対象とし、
  無効化されたアカウントはログインできないようにしました。
- ログイン・登録画面のUIを、既存の一覧・登録ページとデザインを統一し、
  TriArkのブランドカラー（navy #1A2E3A / cyan #00B4C8 / orange #FF8C42）で揃えました。
- ユーザーテーブルは kanri_flg・life_flg に DEFAULT 0 を設定し、
  登録処理での必須項目エラーを防ぐようにしました。

## ⑦フリー項目（感想、シェアしたいこと等なんでも）
今回はハッシュ化とセッションを使ったログイン機能に加え、開発環境そのものの
トラブル対応も経験しました。特に、XAMPPを再インストールした際に htdocs を
上書きしてしまい、全ページが 403 Access forbidden エラーになる事態に遭遇しました。
原因がフォルダのアクセス権限にあると切り分け、chmod でパーミッションを
再付与することで復旧できました。コードのエラーだけでなく、サーバー環境の
権限管理までトラブルシューティングできたのは、実践的で大きな学びでした。
また、500エラーが出るたびに display_errors でエラー内容を表示させ、
「name が必須」「life_flg が必須」といったメッセージを1つずつ読み解いて
対応していく進め方が身につきました。
